# ST-021 — Team section — assigned people with role and domain

**Story ID:** ST-021
**Epic:** EP-04 — Project Detail View
**Priority:** P1
**Points:** 3
**Status:** Not Started

---

## User Story

As a PM or CTO/CEO, I need to see the team currently assigned to a project with their roles and relevant domain knowledge, so that I can assess whether the right skills are on the project.

---

## Acceptance Criteria

- [ ] Team section lists all employees currently assigned to the project
- [ ] Each row shows: employee name, role, domain knowledge list (all domains, not filtered)
- [ ] If no one is assigned — section shows "No team members assigned"
- [ ] Team data is read-only on this view — assignment actions are in EP-05
- [ ] Section visible to both CTO/CEO and PM

---

## Notes

- Depends on EP-05 for the assignment relationship to exist
- Domain knowledge mismatch flag (if any) is shown in this section — logic from EP-05 ST-025
